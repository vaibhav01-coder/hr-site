
--- TABLE 1 ---
Layer	Technology
Frontend	React.js + Tailwind CSS
Backend	Node.js + Express.js (pure — no NestJS, no Fastify)
Database	Supabase — PostgreSQL (@supabase/supabase-js)
Auth	Supabase Auth — OTP via mobile
File Storage	Supabase Storage Bucket — resume PDF
Payment	Stripe — INR, ₹50 = 5000 paise
Email	Nodemailer + Gmail SMTP
Hosting Frontend	Vercel
Hosting Backend	Render or Railway
Environment	.env file — all secrets

--- TABLE 2 ---
Method + Endpoint	What It Does	Auth Required
GET  /api/jobs	Get all active job listings	No
GET  /api/jobs/:id	Get single job details	No
POST /api/jobs	Create new job listing	HR Only
POST /api/candidates/register	Register new candidate	No
GET  /api/candidates/:id	Get candidate profile	No
POST /api/applications	Submit application after payment	No
GET  /api/applications	Get all applications	HR Only
PATCH /api/applications/:id/status	Update application status	HR Only
POST /api/payment/create-intent	Create ₹50 Stripe payment intent	No
POST /api/payment/verify	Verify Stripe payment success	No
POST /api/payment/refund/:id	Refund ₹50 to rejected candidate	HR Only
POST /api/upload/resume	Upload resume PDF to Supabase Storage	No

--- TABLE 3 ---
Step	Action
Step 1	Candidate fills form, clicks Pay ₹50
Step 2	Frontend calls POST /api/payment/create-intent
Step 3	Backend creates Stripe PaymentIntent (5000 paise, INR)
Step 4	Frontend receives clientSecret, opens Stripe Elements UI
Step 5	Candidate enters card details and confirms payment
Step 6	Frontend calls POST /api/payment/verify with intentId
Step 7	Backend confirms status === 'succeeded'
Step 8	Submit button unlocks — candidate submits application
Step 9	Application saved to Supabase with payment_status = paid
Refund	HR marks rejected → POST /api/payment/refund/:id → Stripe refunds ₹50 in 5–7 days
